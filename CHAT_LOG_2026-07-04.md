## 2. 카카오/페이스북 로그인

**사용자 요청:** 카카오톡, 페이스북으로 가입 가능하게 해달라.

- Kakao Developers에서 신규 앱 **"Cam Nang Han" (App ID 1503526)** 생성, 카테고리 라이프스타일, 대표 도메인 `cam-nang-han.vercel.app`
- 플랫폼 키 확인: REST API 키, JavaScript 키, 네이티브 앱 키(사용 안 함) — Cẩm Nang Hàn은 웹 서비스라 REST API 키(Supabase OAuth 클라이언트 ID/시크릿용) + JavaScript 키(추후 카카오맵용)만 필요
- REST API 키 플랫폼 설정: 카카오 로그인 리다이렉트 URI `https://wsvxkamvxqrtothpcety.supabase.co/auth/v1/callback` 등록, 클라이언트 시크릿 활성화
- JavaScript 키 플랫폼 설정: SDK 도메인에 `https://cam-nang-han.vercel.app` + `http://localhost:3118` 등록
- 카카오 로그인 제품 활성화, 동의항목 설정: 닉네임(필수), 프로필 사진(선택), **카카오계정 이메일(필수)** — 이메일 항목을 처음에 빠뜨려 `KOE205` 에러(`잘못된 요청 — 설정하지 않은 동의 항목: account_email`) 발생, 재설정으로 해결
- Supabase Auth → Kakao 프로바이더에 REST API 키 + Client Secret 등록, 활성화
- Supabase Auth → URL Configuration: Site URL을 기본값(`localhost:3000`)에서 `https://cam-nang-han.vercel.app`로 수정, Redirect URLs에 `.../auth/callback`(프로덕션+로컬) 2개 추가
- **실제 카카오 계정(sinabro7500@naver.com)으로 로그인 테스트 성공 확인**

**참고:** 앱 아이콘도 이때 128px PNG로 카카오 콘솔에 업로드(`public/icons/kakao-app-icon-128.png`, git 미커밋 — 로컬 전용).

## 3. 카카오맵 연동 시도 → 부분 성공(폴백 모드)

**사용자 요청:** 카카오맵도 연동해서 위치 기반으로 주변 찾기가 되게 해달라.

- Kakao Developers → 카카오맵 제품 활성화 시도 → **"카카오맵 권한이 없습니다"** 에러, `[앱 설정]>[앱]>[추가 기능 신청]`에서 요청 필요
- 추가 기능 신청 페이지 확인 → **비즈 앱 전환 필요**(사업자 등록 또는 개인 개발자 본인인증+약관동의), 즉시 승인 아님
- 이 상태에서 Maps JS SDK 로드 시도 → `NotAuthorizedError: App(Cam Nang Han) disabled OPEN_MAP_AND_LOCAL service` 확인 (curl로도 재현)
- **REST 로컬 검색 API도 같은 서비스 플래그에 묶여 있어 현재는 사용 불가** (taesil-app은 별도로 이미 비즈 앱 전환된 카카오 앱 키를 쓰는 것으로 추정)
- `app/ban-do/page.tsx` + `components/KakaoMap.tsx` 구현: SDK 로드 성공 시 실제 임베드 지도(카테고리 버튼, 마커, 리스트, 길찾기), **SDK 실패 시 자동으로 `map.kakao.com/link/search/...` 링크 목록 폴백**으로 전환 — 지금은 이 폴백 모드로 서비스 중
- 카테고리: 병원/약국/은행/출입국사무소/주민센터/보건소/다문화가족지원센터/경찰서 (+ 다음 세션에서 카페/술집 2개 추가)
- Vercel에 `NEXT_PUBLIC_KAKAO_JS_KEY` 환경변수 등록(Production/Development)

**남은 작업(다음에 이어갈 것):** 비즈 앱 전환(사용자 직접 — 본인인증 필요) → 승인 후 카카오맵 추가 기능 신청 → 승인되면 `/ban-do`가 자동으로 임베드 지도 모드로 전환됨(코드 수정 불필요, SDK 로드 성공 여부로 자동 분기).

## 5. "Kết bạn gần bạn" — 주변 친구 만들기 기능 추가

**사용자 요청:** taesil-app의 기능을 Cẩm Nang Hàn에 가져와서 합쳐달라. "외국생활 힘드니 가까운 친구들끼리 한잔 할 수도 있고, 모르는 사람이랑 가까우면 친해질 수도 있으니." PWA에도 정상 반영, **API/Supabase는 Cẩm Nang Hàn 자체 것을 사용**.

⚠️ 요청 전 우려사항 전달: 타겟 유저에 미성년 자녀 가족·임산부·이주노동자 등 취약계층 포함되어 있어 데이팅+모텔 성격 기능이 신뢰도/안전 문제를 일으킬 수 있음을 안내 → 사용자가 "친구 만들기" 취지로 재확인, 진행 요청.

### 구현 (데이팅 앱 톤을 배제하고 "친구 만들기 + 안전수칙"으로 설계)
- **Supabase 마이그레이션** (브라우저로 SQL Editor에서 직접 실행, MCP는 다른 계정이라 접근 불가):
  - `supabase/migrations/0002_ket_ban_gan_day.sql` — `profiles`에 `bio/lat/lng/last_seen/visible_nearby`(기본 false, opt-in) 추가, `friend_greetings` 테이블(RLS: 본인이 보낸/받은 것만 조회), `nearby_friends()` RPC(SECURITY DEFINER, 24시간 이내 활동 + 반경 내만, haversine 공식으로 거리 계산, 안전한 필드만 반환)
  - `supabase/migrations/0003_ket_ban_matched.sql` — `matched_friends()` RPC(양방향 인사 교환된 경우만 반환)
- **`app/ket-ban/page.tsx` + `components/ketban/KetBanClient.tsx`**:
  - 안전 수칙 박스 항상 노출(공개 장소에서 만나기, 지인에게 알리기, 송금 금지, 이상하면 바로 이탈)
  - 로그인 필요, opt-in 토글(기본 꺼짐), bio 입력, 위치 켜면 `profiles` upsert + `nearby_friends` 조회
  - "Chào bạn 👋"(인사하기) → `friend_greetings` insert → 양쪽 다 보내면 "🎉 Đã kết nối" 매칭 표시
  - 언제든 "Tắt hiển thị"(끄기) 가능
- `components/KakaoMap.tsx`의 `/ban-do` 카테고리에 **카페(☕)/술집(🍻)** 2개 추가 — "만날 장소 추천"은 기존 지도 인프라 재사용(비즈 앱 승인 전까지는 이것도 폴백 링크로 동작)
- 하단 탭바에 "Kết bạn" 추가(대신 "Ứng dụng"은 탭바에서 빼고 홈/푸터로 이동 — 5탭 유지), 홈페이지 카드, 푸터 nav, sitemap, PWA 서비스워커 프리캐시(버전 v3) 반영

### 검증
- `npm run build` 성공, 로컬 프로덕션 서버에서 **실제 카카오 로그인 계정으로 전체 플로우 테스트**: 위치 켜기 → Supabase `profiles` 테이블에 lat/lng/bio/visible_nearby 정상 저장 확인(SQL로 직접 조회) → `nearby_friends` RPC 정상 응답(다른 opt-in 유저 없어 빈 목록, 정상 동작) → 테스트 데이터 정리(SQL로 원복)
- 커밋 후 push, Vercel 프로덕션 배포 확인(`/ket-ban` 200, 로그인 세션 유지된 채 정상 렌더링)
