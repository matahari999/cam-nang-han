  - `lib/supabase/client.ts`, `lib/supabase/server.ts` — 브라우저/서버 Supabase 클라이언트
  - `middleware.ts`(이후 `proxy.ts`로 리네임) — 세션 쿠키 갱신
  - `app/login/page.tsx` — 이메일 로그인/가입 탭 + Kakao(#FEE500)/Facebook(#1877F2) 버튼, 전부 베트남어
  - `app/auth/callback/route.ts`, `app/auth/signout/route.ts`
  - `components/AuthNav.tsx` — 헤더 로그인 상태 표시
  - `ChecklistInteractive.tsx` 수정 — 로그인 시 `checklist_progress` 테이블과 연동(로드/upsert), 비로그인 시 기존 로컬 상태 유지

### 직접 검토 및 수정한 내용
- `npm run build` 시 "middleware deprecated, use proxy" 경고 발견 → `middleware.ts`→`proxy.ts` 리네임 + export 함수명 `middleware`→`proxy` 변경, 재빌드로 경고 해소 확인
- 브라우저로 실제 회원가입 테스트 → 에러 메시지가 "비밀번호 6자 이상"으로 떴으나 실제로는 **Supabase 무료 티어 기본 메일 발송 rate limit(429)**이 원인이었음을 네트워크 요청/curl 직접 호출로 확인
- 에러 메시지가 실제 원인과 무관하게 하드코딩되어 있던 버그를 고쳐 `error.code` 기준으로 정확한 메시지가 뜨도록 수정
- Kakao 버튼 클릭 시 Supabase OAuth authorize 요청까지 정상 도달하고 "provider is not enabled" 응답을 받는 것을 확인 → 코드는 정상, 대시보드 설정만 남음
- 전부 커밋/push, CLAUDE.md 업데이트

### Kakao/Facebook OAuth 활성화 안내 (사용자가 직접 해야 하는 부분)
1. **Kakao**: Kakao Developers에서 앱 생성 → 카카오 로그인 활성화 → Redirect URI `https://wsvxkamvxqrtothpcety.supabase.co/auth/v1/callback` 등록 → REST API 키 확보
2. **Facebook**: Meta for Developers에서 앱 생성 → Facebook 로그인 제품 추가 → 유효한 OAuth 리디렉션 URI에 동일 콜백 등록 → App ID/Secret 확보
3. **Supabase 대시보드**(`/project/wsvxkamvxqrtothpcety/auth/providers`)에서 Kakao/Facebook 각각 키 입력 후 활성화
4. **Authentication → URL Configuration**: Site URL을 `https://cam-nang-han.vercel.app`로, Redirect URLs 허용목록에 `http://localhost:3118/auth/callback` + `https://cam-nang-han.vercel.app/auth/callback` 추가
5. 참고: DB의 `checked_items`는 항목의 `label` 문자열로 저장되므로, 추후 `lib/checklists.ts`의 라벨 문구를 바꾸면 기존 저장 데이터와 매칭이 깨질 수 있음(라벨을 안정적으로 유지할 것)
